# 2022년 5월 12일 오후6시 미니프로젝트가 종료되었다.

오늘은 마무리로 각 유저가 회원가입 후 리뷰를 등록한 후 보이는 메인페이지에서 현재 로그인 중인 유저에게만 삭제 버튼이 보이게끔 추가 하였다.<br>
유저가 리뷰를 등록하면 서버에 GET요청을 보내서 데이터를 불러온 후 이를 페이지에 표시한다. 이때 jinja2를 이용하여 html이 이해할 수 있는 언어를 넘겨줌으로서 <br>
for문을 통해 mongoDB로부터 데이터를 받아올때 받아온 데이터가 현재 로그인한 유저와 닉네임이 같은지 확인하였다. 같으면 백틱 안에 들어있는 포스팅용 카드박스 안에 삭제버튼을 추가유무를 결정하였다.

<pre><code>
if (nickname != '{{ user_info }}') {
             {#권한 없음#}
              bool = false
} else {
             {#권한 있음#}
             bool = true
}
</code></pre>

 문제는 삭제버튼을 추가해서 이 버튼을 통해 포스팅을 삭제하는 부분에서 시간이 걸렸다. <br>
 삭제 자체는 문자가 이 버튼을 눌렀을 때 어떻게 이 포스팅만 선택해서 지울것이며 삭제 후 포스팅들의 DB처리도 신경쓰였다.<br>
 이를 해결하기 위해 생각한 결과 기존에 회원가입시 아이디와 패스워드 닉네임만 db에 저장햇는데 추가로 포스트개수라는 데이터도 추가해줬다.<br>
 리뷰등록을 하면 이 데이터는 1씩 증가한다. 로그인한 본인한테만 보이는 삭제버튼에는 첫 생성한 포스팅에 1번부터 시작하는 번호를 달아주어서 각각 삭제버튼이 추가해준 포스트넘버랑 일치하게끔
 설정하였다. <br>
 삭제버튼을 누르면 포스트 번호를 클라이언트가 넘겨주고 서버는 이를받아서 닉네임과 포스트 번호가 일치하는 문서를 찾기 시작한다. 여기서 첫번째 문제가 발생한게
 데이터베이스에서 두가지 조건을 만족하는 문서를 찾는 방법을 몰랐던것이다. <br>
 검색을 해봐도 한가지 조건에 대한 검색방법 밖에 몰랐기에 질문하기 전에는 혼자서 찾아보느라 고생했다. <br>
 
 그래도 pymongo관련 CRUD(Create Read Update Delete)문법에 대해 조금이지만 알게되었다.<br>
 닉네임과 삭제버튼에서 받은 포스팅넘버가 같은 게시물을 삭제하는데는 성공햇는데 두번째 문제는 삭제후의 처리였다. <br>
 삭제후에는 1부터 순번대로 배정받은 포스팅 넘버를 삭제한 게시물보다 작은 번호는 괜찮지만 큰 번호는 1씩 줄여줘야 할 필요가 있었다.<br>
 생각하다 넣은 방법으로 문서중에 로그인한 유저의 닉네임으로 작성한 문서에서 포스트 번호를 받아와서 삭제된 포스트 번호보다 큰 경우 -1씩 하도록 for문을 돌렸다.
<pre><code>
        # 닉네임이 일치하는 포스트 번호를 리스트로 받아온다.
        doc = list(db.review.find({'nickname': nickname}, {'numofpost': 1, '_id': False}))

        # for문을 유저의 전체 포스팅 개수만큼 돌면서 삭제한 포스팅 넘버보다 큰 경우 포스팅넘버를 1씩 마이너스 시킨다.
        for i in range(totalpostnum):
            if doc[i]['numofpost'] > int(postnum_receive):
                num = (doc[i]['numofpost'])
                print(num)
                # 포스트 넘버를 -1시켜서 업데이트
                db.review.update_one({'nickname': nickname} and {'numofpost':num}, {'$set': {'numofpost': num-1}})
</code></pre>
 이걸로 특정 번호의 포스팅을 삭제후에도 포스팅 후의 남아있는 포스트번호들이 알맞게 재배치 되어 있는것을 체크했다.<br>
 근데 또 문제가 발생한게 혼자서 게시물을 등록하고 지우는 식으로 테스트를 햇었어서 몰랐는데 이것을 aws ec2에 올린후 팀원들에게 테스트를 부탁햇더니<br>
 예를들어 1번 포스팅을 삭제한다하면 자신의 포스팅이 아닌 다른유저의 1번 포스팅이 삭제되는 문제가 발생하였다. <br>
 코드를 체크해봤는데
 <pre><code>
  # int(postnum_receive)는 삭제 버튼으로부터 받은 포스팅넘버
  db.review.delete_one({'nickname': nickname} and {'numofpost': int(postnum_receive)})
</code></pre>
 위의 코드가 닉네임이 일치하면서 포스팅 번호도 같은 포스팅을 지우는 문법이라고 생각햇는데 이곳에서 뭔가 오동작을 일으키고 있는것 같다.
 아직 해결법을 찾지 못햇지만 아마 delete_one의 사용법이 잘못 되지 않았나 싶다.<br>
 프로젝트를 진행하며 추가적으로 겪었던 문제가 있는데 삭제버튼에서 id를 넘겨주는데 이것이 string형태로 오는지 모르고 {'numofpost': 삭제해야할 포스트번호}<br>
 그대로 썻다가 업데이트나 삭제가 안되서 4시간은 삽질을 햇던거같다. 결국 왜 안돼지 하다가 type함수의 존재를 알게되어서 잘못된 데이터 형식을 넣어주고 있다는 것을 깨닫고
 고칠 수 있었다.<br>
 이외에도 로컬호스트에서는 포스팅시간이 정확히 표시되었는데 어째서인지 aws ec2에 배포하고 나서는 hour부분이 정확히 표시되지 않는 문제가 있었는데 이에 대해서는
 튜터님에게 타임존에 대해 알아보라고 조언을 받았기에 검색해보려한다.
